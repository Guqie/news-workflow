## 优化1：关键词策略优化

### 当前问题
- 配置了55个关键词，但只用了4个
- 如果全用，可能采集过多（500-800条）
- 需要智能选择关键词

### 优化方案

#### 方案A：分层关键词策略
```python
keywords_config = {
    "tier1": ["战略新兴产业", "新能源", "新材料", "数字经济"],  # 核心关键词
    "tier2": ["光伏", "风电", "储能", "人工智能"],              # 细分领域
    "tier3": ["氢能", "石墨烯", "量子计算"]                     # 专业领域
}

# 每日轮换策略
day_of_week = datetime.now().weekday()
if day_of_week < 4:
    keywords = tier1 + random.sample(tier2, 2)  # 核心+2个细分
else:
    keywords = tier1 + random.sample(tier3, 2)  # 核心+2个专业
```

#### 方案B：热度优先策略
- 根据历史采集数量调整
- 热门关键词减少频率
- 冷门关键词增加频率

#### 方案C：目标数量控制
```python
target_news_count = 200  # 目标采集数量
keywords_used = []
total_count = 0

for keyword in keywords:
    if total_count >= target_news_count:
        break
    news = fetch_news(keyword, limit=50)
    keywords_used.append(keyword)
    total_count += len(news)
```
