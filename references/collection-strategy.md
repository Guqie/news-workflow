# 新闻收集方案

## 一、收集策略总览

### 1.1 三层收集架构

```
第一层：RSS订阅（主要）
  ↓ 稳定、结构化、易维护
  
第二层：网页爬取（补充）
  ↓ 覆盖无RSS的网站
  
第三层：搜索引擎（兜底）
  ↓ 确保数量达标
```

### 1.2 收集频率

- **执行时间**: 每天早上 8:00
- **教育人才**: 目标15条，筛选后保留6+条
- **医疗健康**: 目标20条，筛选后保留8+条

---

## 二、方案一：RSS订阅（推荐）

### 2.1 优势

- ✅ 最稳定（不受网站改版影响）
- ✅ 最简单（标准化格式）
- ✅ 最高效（无需解析HTML）
- ✅ 最合规（官方提供的接口）


### 2.2 RSS源清单

#### 教育人才板块
- 人民网教育频道: http://edu.people.com.cn/rss/education.xml
- 新华网教育频道: http://www.xinhuanet.com/edu/news_edu.xml
- 中国教育新闻网: http://www.jyb.cn/rss/jyb.xml

#### 医疗健康板块
- 人民网健康频道: http://health.people.com.cn/rss/health.xml
- 健康报: http://www.jkb.com.cn/rss/jkb.xml

### 2.3 实现方式

使用Python的 `feedparser` 库：

```python
import feedparser

feed = feedparser.parse('http://edu.people.com.cn/rss/education.xml')
for entry in feed.entries:
    title = entry.title
    link = entry.link
    published = entry.published
```

## 三、方案二：网页爬取

### 3.1 适用场景

- 目标网站没有RSS
- 需要特定栏目的内容
- RSS更新不及时

### 3.2 爬取策略

#### 策略A：列表页爬取
```
访问新闻列表页 → 提取标题和链接 → 访问详情页 → 提取正文
```

#### 策略B：搜索结果爬取
```
构造搜索URL → 提取结果列表 → 过滤相关内容
```


### 3.3 反爬虫应对

- 设置合理的请求间隔（1-3秒）
- 使用随机User-Agent
- 必要时使用代理IP
- 遵守robots.txt规则

---

## 四、方案三：搜索引擎

### 4.1 适用场景

- RSS和爬取都无法满足数量要求
- 需要快速补充内容
- 作为兜底方案

### 4.2 搜索策略

#### 百度新闻搜索
```
https://www.baidu.com/s?tn=news&word=关键词
```

#### 今日头条搜索
```
https://www.toutiao.com/search/?keyword=关键词
```


### 4.3 关键词策略

#### 教育人才板块关键词
- 一级关键词：教育政策、人才引进、高校改革
- 二级关键词：职业教育、招生考试、就业市场

#### 医疗健康板块关键词
- 一级关键词：医疗政策、医药监管、医院改革
- 二级关键词：生物医药、医疗器械、健康产业
